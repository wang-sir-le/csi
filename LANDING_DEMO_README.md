# CSI 人体状态识别落地演示说明

## 当前可演示功能

系统已具备从 ESP32 接收端实时读取 CSI 数据，并输出人体状态预测结果的能力。当前支持三类状态：

1. 无人
2. 静止
3. 翻滚

当前模型基于 16 组采集数据训练：

| 类别 | 采集组 |
|---|---|
| 静止 | group_01 至 group_06 |
| 翻滚 | group_07 至 group_11 |
| 无人 | group_12 至 group_16 |

## 一键运行实时演示

确认两块 ESP32 均已上电，接收端串口为 `COM13`，然后在当前目录运行：

```powershell
powershell -ExecutionPolicy Bypass -File .\start_realtime_demo.ps1
```

终端会持续输出类似结果：

```text
[   3.2s] packets=  128 pred=无人 stable=无人 probs=(无人:0.78, 翻滚:0.22, 静止:0.00)
```

字段说明：

| 字段 | 含义 |
|---|---|
| packets | 已接收有效 CSI 包数 |
| pred | 当前窗口预测状态 |
| stable | 多窗口投票后的稳定状态 |
| probs | 三类预测概率 |

停止演示时按 `Ctrl+C`。

## 一键运行 Web 看板

若需要更适合展示的页面界面，运行：

```powershell
powershell -ExecutionPolicy Bypass -File .\start_web_dashboard.ps1
```

然后在浏览器访问：

```text
http://127.0.0.1:8000
```

页面会显示：

1. 当前稳定状态；
2. 当前窗口预测状态；
3. 无人、静止、翻滚三类概率；
4. 有效 CSI 包数；
5. 实时预测日志。

## 重新训练模型

当新增训练数据或修改标签后，运行：

```powershell
powershell -ExecutionPolicy Bypass -File .\run_multi_group_pipeline.ps1
```

该脚本会重新生成质量统计、特征表、图表、模型指标，并保存最新模型：

```text
data/multi_group/models/csi_state_classifier.joblib
```

## 当前模型指标

当前训练脚本采用 `GroupKFold`，即按采集组划分训练集和测试集，避免同一组相邻窗口同时进入训练和测试造成泄漏。

| 指标 | 数值 |
|---|---:|
| Accuracy | 1.000 |
| Macro F1-score | 1.000 |
| 类别数 | 3 |
| 窗口样本数 | 270 |
| 采集组数 | 16 |

## 演示注意事项

1. 演示前不要移动两块 ESP32 的位置。
2. 无人状态下，监测区域内尽量不要有人靠近或经过。
3. 静止状态下，人体应保持在训练时相近的位置。
4. 翻滚状态下，动作幅度应与采集训练数据时接近。
5. 若输出“不确定”，说明当前窗口最高预测概率低于阈值，可等待下一窗口或调整姿态。

## 当前限制

1. 当前数据来自同一环境，跨房间、跨距离、跨人员泛化能力尚未验证。
2. 目前是人体状态识别，不是完整睡眠分期系统。
3. 若要扩展到睡眠呼吸监测，需要继续采集静止呼吸微动数据，并增加呼吸频率标注或参考设备数据。
